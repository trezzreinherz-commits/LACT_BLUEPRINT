# Purpose

Dokumen ini mendefinisikan arsitektur data LACT.

Dokumen ini **bukan sekadar daftar tabel**, tetapi menjelaskan bagaimana data mengalir di dalam sistem.

Seluruh modul pada LACT dibangun mengikuti struktur data yang dijelaskan pada dokumen ini.

Database merupakan **Single Source of Truth**.

---

# Database Philosophy

LACT dibangun menggunakan pendekatan **Data-Centric Architecture**.

Artinya:

* Module mengikuti data.
* UI mengikuti data.
* Document mengikuti data.
* API mengikuti data.

Bukan sebaliknya.

---

# Database Hierarchy

```text
MASTER DATA
      │
      ▼
PROJECT
      │
      ▼
TRANSACTION DATA
      │
      ▼
REFERENCE DATA
      │
      ▼
OUTPUT DATA
```

Setiap lapisan hanya boleh membaca atau menulis data sesuai tanggung jawabnya.

---

# MASTER DATA

Master Data adalah data permanen yang hanya dapat diubah oleh developer.

User tidak memiliki hak untuk mengubah data ini.

---

## khs_qe

Purpose

Database referensi pekerjaan.

Digunakan untuk:

* Deskripsi
* Unit
* Paket Material
* Paket Jasa

Karakteristik

✔ Read Only

✔ Tidak berubah selama operasional

✔ Menjadi referensi seluruh perhitungan BOQ

---

# PROJECT DATA

Project merupakan root dari seluruh data.

Semua tabel transaksi wajib memiliki relasi terhadap Project.

---

## projects

Purpose

Menyimpan identitas proyek.

Primary Key

id

Unique Identifier

id_ihld

Rules

Semua tabel transaksi wajib memiliki:

project_id

yang mengarah ke:

projects.id

Project adalah induk seluruh data.

---

# TRANSACTION DATA

Transaction Data adalah data yang dibuat atau diubah oleh user.

Data inilah yang menjadi pusat operasional LACT.

---

## boq_items

Purpose

Pusat transaksi pekerjaan proyek.

Source

Upload BOQ

Input User

Update User

Reference

khs_qe

Business Engine

Nilai DRM

Nilai Actual

Designator Normalization

Data yang sering digunakan

* Designator
* Actual

Digunakan oleh

* ODP
* Evidence
* Summary
* Document Generator

Rules

BOQ merupakan pusat workflow proyek.

---

## opm_data

Purpose

Menyimpan seluruh data pengukuran OPM.

Source

Input User.

Digunakan oleh

* OPM Table
* Catuan
* Summary
* Document Generator

Rules

Seluruh informasi utama OPM berada pada tabel ini.

---

## galian_inspections

Purpose

Menyimpan seluruh data inspeksi Galian.

Karakteristik

Header Transaction.

Digunakan untuk menghasilkan tabel Galian.

---

## evidences

Purpose

Metadata seluruh evidence proyek.

Source

Upload User.

Rules

Tidak menyimpan logika bisnis.

Hanya metadata.

---

# REFERENCE DATA

Reference Data merupakan data turunan dari Transaction Data.

Data ini dibentuk otomatis oleh sistem atau digunakan sebagai pendukung tampilan.

---

## odp_tabs

Source

boq_items

Menggunakan:

Designator

Actual

Purpose

Membuat struktur ODP secara otomatis.

Rules

ODP bukan sumber data.

ODP hanya representasi data BOQ.

---

## opm_ports

Source

opm_data

Purpose

Menyimpan data setiap port OPM.

Rules

Tidak berdiri sendiri.

Harus memiliki relasi dengan opm_data.

---

## catuan

Source

opm_data

Purpose

Menyimpan data catuan yang digunakan pada tabel OPM.

Rules

Seluruh data berasal dari opm_data.

---

# OUTPUT DATA

Output Data merupakan hasil akhir sistem.

Output tidak boleh menjadi sumber data.

---

## generated_documents

Purpose

Metadata dokumen hasil generate.

Rules

Hanya output.

Tidak boleh digunakan sebagai input modul lain.

# Normalization Rule

Designator yang berasal dari BOQ harus melalui proses normalisasi sebelum digunakan oleh modul lain.

Seluruh pencarian Designator menggunakan hasil normalisasi.

Tidak diperbolehkan melakukan normalisasi berbeda pada setiap modul.
